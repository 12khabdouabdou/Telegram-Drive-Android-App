# Telegram-Drive Agent Instructions

## Repository Architecture
- **Framework:** Tauri v2 application (cross-platform desktop & Android).
- **Frontend:** React, TypeScript, TailwindCSS (v4, `@tailwindcss/postcss`), Vite, Framer Motion (in `app/`).
- **Backend:** Rust, `grammers` (Telegram API client), Tokio, Actix-web (streaming server + REST API), SQLite (in `app/src-tauri/`).

## Development Commands
- **Install:** `cd app && npm install`
- **Desktop Dev:** `cd app && npm run tauri dev`
- **Desktop Build:** `cd app && npm run tauri build` (runs `tsc && vite build` first)
- **Android Dev/Build:**
  - *Must run first:* `cd app && npm run tauri android init`
  - *Then build:* `cd app && npm run tauri android build`
- **Rust Tests:** `cd app/src-tauri && cargo test` (platform-agnostic tests only; Android tests are `#[cfg(target_os = "android")]` and need an emulator)
- **Frontend Typecheck:** `cd app && npx tsc --noEmit`
- **Lint:** None configured.

## Key Structural Notes
- **Two capabilities files:** `app/src-tauri/capabilities/default.json` (desktop) and `mobile.json` (Android-only). Add new Tauri permissions in the right file.
- **Two context directories:** `app/src/context/` (Confirm, Settings, Theme) and `app/src/contexts/` (DropZone). Both are active; check both when modifying context providers.
- **Tauri plugins in use:** store, shell, dialog, updater, fs, process, deep-link, clipboard-manager, window-state, opener, os. The updater plugin is excluded on Android (`#[cfg(not(target_os = "android"))]` in `lib.rs:365`).
- **Dev server ports:** Vite on 1420, Actix media streaming on 14201 (const `STREAM_PORT` in `lib.rs:66`). CSP in `tauri.conf.json` allows these.
- **Frontend entry:** `app/src/main.tsx` → `App.tsx` → lazy-loaded `DesktopDashboard` / `MobileDashboard`.

## Critical Android Quirks
- `app/src-tauri/gen/android/` is gitignored (`/app/src-tauri/gen/` in `.gitignore`).
- **Native Android Code (Kotlin):** Custom files (e.g., `AutoBackupService.kt`, `MainActivity.kt`) are tracked with `git add -f`.
- **AndroidManifest.xml:** Not tracked locally. Generated by `tauri android init` and patched via `sed` in `.github/workflows/android.yml`. Add permission/intent changes as `sed` patches there.
- **Zero-Copy Uploads:** Frontend passes `content://` URIs. Rust converts these to `fd://` via JNI `ParcelFileDescriptor`. *Security:* frontend must never pass `fd://` directly.
- **Share-to-App flow:** Android shares → Kotlin caches URIs → Rust `share_intent.rs` reads them → React pulls via `cmd_get_pending_shared_files` on mount.

## Rust Backend & JNI Quirks
- **Tauri v2 Traits:** Explicitly import `use tauri::Emitter;` for `.emit()` and `use tauri::Manager;` for `.try_state()` / `.state()`.
- **JNI String Borrowing (E0597):** Don't assign `env.get_string(&jstr)` to a temp inside an `if let` if the `jstr` is created in the same scope. Consume instantly: `env.get_string(&jstr).map(|s| s.to_string_lossy().into_owned())`.
- **JNI Exceptions:** Don't use `?` after a JNI call that may throw. Call `env.exception_clear()` before returning `Err` across FFI or the JVM crashes.
- **`TelegramState` runner_shutdown:** Must shut down the old grammers runner before spawning a new one on reconnect, or runner tasks accumulate and cause stack overflow (`commands/mod.rs:10`).
- **Linux:** `main.rs` sets `WEBKIT_DISABLE_DMABUF_RENDERER=1` before `tauri::Builder` init to fix `EGL_BAD_ALLOC` on Arch/AppImages.
- **Windows:** `lib.rs` calls `CoInitializeEx` with `COINIT_MULTITHREADED` on worker threads that touch COM (Actix, Tokio networking) to prevent `RPC_E_CHANGED_MODE`.
- **Android:** Actix streaming server is disabled on Android (`#[cfg(not(target_os = "android"))]`). JNI init uses `catch_unwind` to prevent panics from crossing the FFI boundary.

## React Frontend Quirks
- **Tauri Store I/O Spam:** When listening to `upload-progress` / `download-progress`, do NOT put the raw `queue` array in the dependency list for `store.save()` effects (4 ticks/s causes extreme I/O). Filter to only trigger on `id` or `status` changes.
- **Routing:** State-based (`activeTab`, `authStatus`) — no formal router.
- **Virtualization:** `TouchFileList.tsx` uses `@tanstack/react-virtual`. Scroll container is `id="mobile-scroll-container"` in `MobileDashboard.tsx`.
- **Update flow:** `UpdateBanner` component in `App.tsx` is wired via `useUpdateCheck` hook; requires `process:allow-restart` capability.
- **Strict TypeScript:** `tsconfig.json` has `noUnusedLocals` and `noUnusedParameters` — these will fail `tsc`/build.
- **Build command:** `npm run build` runs `tsc && vite build` (not just `vite build`).

## Testing
- **Rust backend:** `cargo test` runs platform-agnostic URI cache tests in `app/src-tauri/tests/`. Android-targeted tests exist but only compile/test on actual Android targets (`#[cfg(target_os = "android")]`).
- **Frontend:** No automated tests configured.
